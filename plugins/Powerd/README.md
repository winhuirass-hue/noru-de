# Powerd Plugin

Qt 6.8.2 port of the Powerd plugin from Lomiri. Provides QML access to system display power state via DBus/Unity.Screen.

**Qt Version**: 6.8.2 (required)  
**Platform**: Ubuntu 24.04.3 LTS

---

## Overview

Controls system display power state including turning display on/off, monitoring power state changes, and managing display timeouts through Powerd daemon via DBus.

---

## Build

### Requirements

- Qt 6.8.2
- CMake 3.17+
- Linux with DBus/Powerd
- GLib 2.0 and GIO 2.0

### Build Plugin

```bash
cd plugins/Powerd
mkdir build && cd build
cmake .. && make
```

Output: `libPowerd.so`, `libPowerdplugin.so`, `qmldir`

### Build QML Tests

```bash
cd plugins/Powerd/Tests/QMLTests
mkdir build && cd build
cmake .. && make
./appQMLTests
```

### Build Unit Tests

```bash
cd plugins/Powerd/Tests/UnitTests
mkdir build && cd build
cmake .. && make
ctest --verbose
```

## API

### Properties (Read-only, Notifiable)

| Property | Type | Description |
|----------|------|-------------|
| `status` | Status | Display power state (0: Off, 1: On) |

### Methods

| Method | Parameters | Description |
|--------|-----------|-------------|
| `setStatus(status, reason)` | Status, DisplayStateChangeReason | Set display power state |

### Enums

#### Status
```
Off = 0          // Display is powered off
On = 1           // Display is powered on
```

#### DisplayStateChangeReason
```
Unknown = 0      // Unknown reason
Inactivity = 1   // User inactivity timeout
PowerKey = 2     // User pressed power button
Proximity = 3    // Proximity sensor triggered
Notification = 4 // Notification received
SnapDecision = 5 // Snap decision prompt
CallDone = 6     // Voice call ended
```

### Signals

| Signal | Parameter | Description |
|--------|-----------|-------------|
| `statusChanged` | DisplayStateChangeReason | Emitted when display power state changes |

## Qt5 to Qt6 Migration

### CMakeLists.txt

**Key Changes:**

- Replaced `add_library(MODULE)` + `add_lomiri_plugin()` with `qt_add_qml_module()`
- Changed `find_package(Qt5 ...)` to `find_package(Qt6 ...)`
- Changed `target_link_libraries()` from `Qt5::` to `Qt6::` targets with `PRIVATE` keyword
- `qt_add_qml_module()` automatically generates `qmldir`

### Header Files

**Pattern (applies to all exposed classes):**

**Qt5 (removed plugin factory code):**
```cpp
// plugin.cpp (DELETED)
void SomePlugin::registerTypes(const char *uri) {
    qmlRegisterType<ClassName>(uri, 1, 0, "ClassName");
    qmlRegisterSingletonType<SingletonClass>(uri, 1, 0, "SingletonClass", ...);
    qmlRegisterUncreatableType<ReadOnlyData>(uri, 1, 0, "ReadOnlyData", "ReadOnlyData cannot be created from QML");
}
```

**Qt6 (macro-based registration):**
```cpp
// Each class header
#include <qqmlintegration.h>

// Regular creatable type
class ClassName: public QObject {
    Q_OBJECT
    QML_ELEMENT     // Can be instantiated from QML
    // ...
};

// Singleton type
class SingletonClass: public QObject {
    Q_OBJECT
    QML_ELEMENT
    QML_SINGLETON   // Only one instance, accessible globally
    // ...
};

// Uncreatable type (accessible but cannot be instantiated from QML)
class ReadOnlyData: public QObject {
    Q_OBJECT
    QML_ELEMENT
    QML_UNCREATABLE("ReadOnlyData cannot be created from QML")
    // ...
};
```

**Why**: Qt6 uses compile-time macro-based registration instead of runtime type registration. This eliminates runtime overhead, reduces binary size, and removes the need for plugin factory classes.

### Files Deleted

- `plugin.h` — No longer needed (replaced by `QML_ELEMENT` macro)
- `plugin.cpp` — No longer needed (replaced by `QML_ELEMENT` macro)
- `qmldir` — Auto-generated by `qt_add_qml_module()`

### DBus Changes

**Qt5:**
```cpp
QDBusConnection::SM_BUSNAME()  // Session/System message bus selection
```

**Qt6:**
```cpp
QDBusConnection::systemBus()   // Explicit system bus
```
