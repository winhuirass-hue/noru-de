# ModemConnectivity Plugin

Qt 6 port of the ModemConnectivity plugin from Lomiri. Provides D-Bus/QML access to connectivity services, specifically for unlocking modems.

**Qt Version**: Qt 6.8.2

**Platform**: Ubuntu (Lomiri)

---

## Overview

Provides a simple interface to trigger the unlocking of all modems via a D-Bus signal to `com.lomiri.connectivity1`. This is exposed to QML as a singleton for easy access within the Lomiri shell.

---

## Build

### Requirements

- Qt 6.8.2
- CMake 3.17+
- Linux with D-Bus

### Build Plugin

```bash
cd plugins/Lomiri/ModemConnectivity
rm -rf build
mkdir build && cd build
cmake ..
make
```

Output: `libModemConnectivity.so`, `libModemConnectivityplugin.so`, `qmldir`

### Build QML Tests

```bash
cd plugins/Lomiri/ModemConnectivity/Tests/QMLTests
rm -rf build
mkdir build && cd build
cmake ..
make
./appQMLTests
```

### Build Unit Tests

```bash
cd plugins/Lomiri/ModemConnectivity/Tests/UnitTests
rm -rf build
mkdir build && cd build
cmake ..
make
ctest --verbose
```

## Qt5 to Qt6 Migration

### CMakeLists.txt

**Key Changes:**

- Replaced `add_library(MODULE)` + `add_lomiri_plugin()` with `qt_add_qml_module()`
- Changed `find_package(Qt5 ...)` to `find_package(Qt6 ...)`
- Changed `target_link_libraries()` from `Qt5::` to `Qt6::` targets with `PRIVATE` keyword
- `qt_add_qml_module()` automatically generates `qmldir`

### Header Files

**Pattern (applies to all exposed classes):**
**Qt5 (removed plugin factory code):**
```cpp
// plugin.cpp (DELETED)
void SomePlugin::registerTypes(const char *uri) {
    qmlRegisterSingletonType<ModemConnectivity>(uri, 0, 1, "ModemConnectivity", ...);
}
```

**Qt6 (macro-based registration):**
```cpp
// ModemConnectivity.h
#include <qqmlintegration.h>

class ModemConnectivity : public QObject {
    Q_OBJECT
    QML_ELEMENT
    QML_SINGLETON
    // ...
};
```

**Why**: Qt 6 uses compile-time macro-based registration instead of runtime type registration. This eliminates runtime overhead, reduces binary size, and removes the need for plugin factory classes.

### Files Deleted

- `plugin.h` — No longer needed (replaced by `QML_ELEMENT` macro)
- `plugin.cpp` — No longer needed (replaced by `QML_ELEMENT` macro)
- `qmldir` — Auto-generated by `qt_add_qml_module()`
