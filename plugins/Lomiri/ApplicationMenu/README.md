# ApplicationMenu Plugin

Qt 6.8.2 port of the ApplicationMenu plugin from Lomiri. Provides DBus/QML access to application and surface menus via the GMenu protocol.

**Qt Version**: 6.8.2
 
**Platform**: Ubuntu 24.04.3 LTS

---

## Overview

Manages application and surface menus in Lomiri through a DBus service. Allows applications to register their menus (GMenu protocol) which can then be displayed in the shell's menu bar or other UI elements. Provides QML access to retrieve registered menus for a given surface/window.

---

## Build

### Requirements

- Qt 6.8.2
- CMake 3.17+
- Linux with DBus

### Build Plugin

```bash
cd plugins/Lomiri/ApplicationMenu
mkdir build && cd build
cmake .. && make
```

Output: `libApplicationMenu-qml.so`, `libApplicationMenu-qmlplugin.so`, `qmldir`


### Build Unit Tests

```bash
cd plugins/Lomiri/ApplicationMenu/Tests/UnitTests
```

## API

### MenuServicePath (QML_UNCREATABLE)

**Properties (Read-only):**

| Property | Type | Description |
|----------|------|-------------|
| `service` | QByteArray | DBus service name |
| `menuPath` | QByteArray | DBus path to menu |
| `actionPath` | QByteArray | DBus path to actions |

### DBusApplicationMenuRegistry (QML_SINGLETON)

**Methods:**

| Method | Arguments | Returns | Description |
|--------|-----------|---------|-------------|
| `getMenusForSurface(surfaceId)` | `QString surfaceId` | `QList<QObject*>` | Get all registered menus for a surface |

**Signals:**

| Signal | Arguments | Description |
|--------|-----------|-------------|
| `appMenuRegistered` | `uint pid` | Emitted when app menu registered |
| `appMenuUnregistered` | `uint pid` | Emitted when app menu unregistered |
| `surfaceMenuRegistered` | `QString surfaceId` | Emitted when surface menu registered |
| `surfaceMenuUnregistered` | `QString surfaceId` | Emitted when surface menu unregistered |

## Qt5 to Qt6 Migration

### CMakeLists.txt

**Key Changes:**

- Replaced `add_library(MODULE)` + `add_lomiri_plugin()` with `qt_add_qml_module()`
- Changed `find_package(Qt5 ...)` to `find_package(Qt6 ...)`
- Changed `target_link_libraries()` from `Qt5::` to `Qt6::` targets with `PRIVATE` keyword
- `qt_add_qml_module()` automatically generates `qmldir`

### Header Files

**Pattern (applies to all exposed classes):**

**Qt5 (removed plugin factory code):**
```cpp
// plugin.cpp (DELETED)
void SomePlugin::registerTypes(const char *uri) {
    qmlRegisterType<ClassName>(uri, 1, 0, "ClassName");
    qmlRegisterSingletonType<SingletonClass>(uri, 1, 0, "SingletonClass", ...);
    qmlRegisterUncreatableType<ReadOnlyData>(uri, 1, 0, "ReadOnlyData", "ReadOnlyData cannot be created from QML");
}
```

**Qt6 (macro-based registration):**
```cpp
// Each class header
#include <qqmlintegration.h>

// Regular creatable type
class ClassName: public QObject {
    Q_OBJECT
    QML_ELEMENT     // Can be instantiated from QML
    // ...
};

// Singleton type
class SingletonClass: public QObject {
    Q_OBJECT
    QML_ELEMENT
    QML_SINGLETON   // Only one instance, accessible globally
    // ...
};

// Uncreatable type (accessible but cannot be instantiated from QML)
class ReadOnlyData: public QObject {
    Q_OBJECT
    QML_ELEMENT
    QML_UNCREATABLE("ReadOnlyData cannot be created from QML")
    // ...
};
```


**Why**: Qt6 uses compile-time macro-based registration instead of runtime type registration. This eliminates runtime overhead, reduces binary size, and removes the need for plugin factory classes.

### Files Deleted

- `plugin.h` — No longer needed (replaced by `QML_ELEMENT` macro)
- `plugin.cpp` — No longer needed (replaced by `QML_ELEMENT` macro)
- `qmldir` — Auto-generated by `qt_add_qml_module()`
