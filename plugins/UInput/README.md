# UInput Plugin

Qt 6.8.2 port of the UInput plugin from Lomiri. Provides QML access to virtual input device creation and control via the Linux kernel uinput subsystem.

**Qt Version**: 6.8.2 (required)  
**Platform**: Ubuntu 24.04.3 LTS  
**Privileges**: Requires root access to `/dev/uinput`

---

## Overview

Creates and controls a virtual mouse input device via the Linux kernel's uinput subsystem. Enables synthetic mouse movements, button presses, and scroll wheel events programmatically. This is useful for automated testing, accessibility tools, and input simulation.

---

## Build

### Requirements

- Qt 6.8.2
- CMake 3.17+
- Linux kernel with uinput support
- Root privileges (for runtime testing)

### Build Plugin

```bash
cd plugins/UInput
mkdir build && cd build
cmake .. && make
```

Output: `libUInput.so`, `libUInputplugin.so`, `qmldir`

### Build QML Tests

```bash
cd plugins/UInput/Tests/QMLTests
mkdir build && cd build
cmake .. && make
./appQMLTests
```

### Build Unit Tests

```bash
cd plugins/UInput/Tests/UnitTests
mkdir build && cd build
cmake .. && make
sudo ctest --verbose
```

**Note**: Unit tests require root privileges due to `/dev/uinput` access.

## API

### Enums

| Enum | Values | Description |
|------|--------|-------------|
| `Button` | `ButtonLeft` (0), `ButtonRight` (1), `ButtonMiddle` (2) | Mouse button identifiers |

### Invokable Methods

| Method | Parameters | Returns | Description |
|--------|-----------|---------|-------------|
| `createMouse()` | None | void | Creates virtual mouse device |
| `removeMouse()` | None | void | Destroys virtual mouse device |
| `moveMouse(dx, dy)` | int, int | void | Moves mouse cursor relative to current position |
| `pressMouse(button)` | Button enum | void | Presses mouse button |
| `releaseMouse(button)` | Button enum | void | Releases mouse button |
| `scrollMouse(dh, dv)` | int, int | void | Scrolls with horizontal (dh) and vertical (dv) displacement |

## Qt5 to Qt6 Migration

### CMakeLists.txt

**Key Changes:**

- Replaced `add_library(MODULE)` + `add_lomiri_plugin()` with `qt_add_qml_module()`
- Changed `find_package(Qt5 ...)` to `find_package(Qt6 ...)`
- Changed `target_link_libraries()` from `Qt5::` to `Qt6::` targets with `PRIVATE` keyword
- `qt_add_qml_module()` automatically generates `qmldir`

### Header Files

**Pattern (applies to all exposed classes):**

**Qt5 (removed plugin factory code):**
```cpp
// plugin.cpp (DELETED)
void SomePlugin::registerTypes(const char *uri) {
    qmlRegisterType<ClassName>(uri, 1, 0, "ClassName");
    qmlRegisterSingletonType<SingletonClass>(uri, 1, 0, "SingletonClass", ...);
    qmlRegisterUncreatableType<ReadOnlyData>(uri, 1, 0, "ReadOnlyData", "ReadOnlyData cannot be created from QML");
}
```

**Qt6 (macro-based registration):**
```cpp
// Each class header
#include <qqmlintegration.h>

// Regular creatable type
class ClassName: public QObject {
    Q_OBJECT
    QML_ELEMENT     // Can be instantiated from QML
    // ...
};

// Singleton type
class SingletonClass: public QObject {
    Q_OBJECT
    QML_ELEMENT
    QML_SINGLETON   // Only one instance, accessible globally
    // ...
};

// Uncreatable type (accessible but cannot be instantiated from QML)
class ReadOnlyData: public QObject {
    Q_OBJECT
    QML_ELEMENT
    QML_UNCREATABLE("ReadOnlyData cannot be created from QML")
    // ...
};
```

**Why**: Qt6 uses compile-time macro-based registration instead of runtime type registration. This eliminates runtime overhead, reduces binary size, and removes the need for plugin factory classes.

### Files Deleted

- `plugin.h` — No longer needed (replaced by `QML_ELEMENT` macro)
- `plugin.cpp` — No longer needed (replaced by `QML_ELEMENT` macro)
- `qmldir` — Auto-generated by `qt_add_qml_module()`
