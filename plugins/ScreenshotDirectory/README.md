# ScreenshotDirectory Plugin

Qt 6.8.2 port of the ScreenshotDirectory plugin from Lomiri. Provides QML access to generate unique screenshot filenames with automatic timestamped paths.

**Qt Version**: 6.8.2 (required)  
**Platform**: Ubuntu 24.04.3 LTS

---

## Overview

Generates unique, timestamped screenshot filenames and creates the necessary directory structure. Filenames follow the format: `screenshot_yyyyMMdd_hhmmsszzz.png` where the timestamp ensures each screenshot has a unique filename. Automatically creates the `~/Pictures/Screenshots/` directory structure on initialization.

## Build

### Requirements

- Qt 6.8.2
- CMake 3.17+
- Linux (Ubuntu 24.04.3 LTS)

### Build Plugin

```bash
cd plugins/ScreenshotDirectory
mkdir build && cd build
cmake .. && make
```

Output: `libScreenshotDirectory-qml.so`,`libScreenshotDirectory-qmlplugin.so`, `qmldir`

### Build QML Tests

```bash
cd plugins/ScreenshotDirectory/Tests/QMLTests
mkdir build && cd build
cmake .. && make
./appQMLTests
```

### Build Unit Tests

```bash
cd plugins/ScreenshotDirectory/Tests/UnitTests
mkdir build && cd build
cmake .. && make
ctest --verbose
```

## API

### Methods (Q_INVOKABLE)

| Method | Returns | Description |
|--------|---------|-------------|
| `makeFileName()` | QString | Returns unique screenshot filename with full path: `/path/to/Pictures/Screenshots/screenshot_yyyyMMdd_hhmmsszzz.png` |

## Qt5 to Qt6 Migration

### CMakeLists.txt

**Key Changes:**

- Replaced `add_library(MODULE)` + `add_lomiri_plugin()` with `qt_add_qml_module()`
- Changed `find_package(Qt5 ...)` to `find_package(Qt6 ...)`
- Changed `target_link_libraries()` from `Qt5::` to `Qt6::` targets with `PRIVATE` keyword
- `qt_add_qml_module()` automatically generates `qmldir`

### Header Files

**Pattern (applies to all exposed classes):**

**Qt5 (removed plugin factory code):**
```cpp
// plugin.cpp (DELETED)
void SomePlugin::registerTypes(const char *uri) {
    qmlRegisterType<ClassName>(uri, 1, 0, "ClassName");
    qmlRegisterSingletonType<SingletonClass>(uri, 1, 0, "SingletonClass", ...);
    qmlRegisterUncreatableType<ReadOnlyData>(uri, 1, 0, "ReadOnlyData", "ReadOnlyData cannot be created from QML");
}
```

**Qt6 (macro-based registration):**
```cpp
// Each class header
#include <qqmlintegration.h>

// Regular creatable type
class ClassName: public QObject {
    Q_OBJECT
    QML_ELEMENT     // Can be instantiated from QML
    // ...
};

// Singleton type
class SingletonClass: public QObject {
    Q_OBJECT
    QML_ELEMENT
    QML_SINGLETON   // Only one instance, accessible globally
    // ...
};

// Uncreatable type (accessible but cannot be instantiated from QML)
class ReadOnlyData: public QObject {
    Q_OBJECT
    QML_ELEMENT
    QML_UNCREATABLE("ReadOnlyData cannot be created from QML")
    // ...
};
```

**Why**: Qt6 uses compile-time macro-based registration instead of runtime type registration. This eliminates runtime overhead, reduces binary size, and removes the need for plugin factory classes.

### Files Deleted

- `plugin.h` — No longer needed (replaced by `QML_ELEMENT` macro)
- `plugin.cpp` — No longer needed (replaced by `QML_ELEMENT` macro)
- `qmldir` — Auto-generated by `qt_add_qml_module()`

## Testing

### QML Testing

Test project located in `Tests/QMLTests/`. Automatically discovers plugin build directory and runs tests.

**Tests validate:**
- Plugin can be instantiated in QML
- `makeFileName()` method is callable from QML
- Return value is a valid string path
- Path can be displayed in QML UI

### Unit Tests

Tests validate:
- ✓ Object creation and initialization
- ✓ `makeFileName()` returns non-empty string with path
- ✓ Filename contains "screenshot" prefix
- ✓ Filename contains timestamp in format `yyyyMMdd_hhmmsszzz`
- ✓ Filename ends with `.png` extension
- ✓ Multiple calls produce different filenames (timestamp uniqueness)
- ✓ Directory path exists and is valid
- ✓ Files can actually be created at generated path

**Expected result:**
```
100% tests passed, 0 tests failed
```
