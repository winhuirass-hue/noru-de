# GlobalShortcut Plugin

Qt 6.8.2 port of the GlobalShortcut plugin from Lomiri. Provides a QML component for registering and detecting global keyboard shortcuts with key press and release notifications.

**Qt Version**: 6.8.2 (required)  
**Platform**: Ubuntu 24.04.3 LTS

---

## Overview

A QML item component (`QQuickItem`) that allows registering global keyboard shortcuts. Shortcuts can be specified either as a string (`"Ctrl+Alt+L"`) or as Qt enum values (`Qt.ControlModifier|Qt.AltModifier|Qt.Key_L`). When a registered shortcut key is pressed or released, the corresponding signal (`triggered` / `released`) is emitted. The plugin uses a central `GlobalShortcutRegistry` to manage all registered shortcuts and filters key events at the window level.

## API

### Properties (Read/Write, Notifiable)

| Property | Type | Description |
|----------|------|-------------|
| `shortcut` | QVariant | The shortcut key — Qt enum syntax (e.g., `Qt.Key_L` or `Qt.ControlModifier \| Qt.AltModifier \| Qt.Key_L`) |
| `active` | bool | Whether the shortcut is active (default: `true`) |

### Signals

| Signal | Parameters | Description |
|--------|-----------|-------------|
| `triggered()` | shortcut: string | Emitted when the registered shortcut key is pressed |
| `released()` | shortcut: string | Emitted when the registered shortcut key is released |
| `shortcutChanged()` | shortcut: QVariant | Emitted when the shortcut property value changes |
| `activeChanged()` | active: bool | Emitted when the active property value changes |

## Qt5 to Qt6 Migration

### CMakeLists.txt

**Key Changes:**

- Replaced `add_library(MODULE)` + `add_lomiri_plugin()` with `qt_add_qml_module()`
- Changed `find_package(Qt5 ...)` to `find_package(Qt6 ...)`
- Changed `target_link_libraries()` from `Qt5::` to `Qt6::` targets with `PRIVATE` keyword
- `qt_add_qml_module()` automatically generates `qmldir`

### Header Files

**Pattern (applies to all exposed classes):**

**Qt5 (removed plugin factory code):**
```cpp
// plugin.cpp (DELETED)
void SomePlugin::registerTypes(const char *uri) {
    qmlRegisterType<ClassName>(uri, 1, 0, "ClassName");
    qmlRegisterSingletonType<SingletonClass>(uri, 1, 0, "SingletonClass", ...);
    qmlRegisterUncreatableType<ReadOnlyData>(uri, 1, 0, "ReadOnlyData", "ReadOnlyData cannot be created from QML");
}
```

**Qt6 (macro-based registration):**
```cpp
// Each class header
#include <qqmlintegration.h>

// Regular creatable type
class ClassName: public QObject {
    Q_OBJECT
    QML_ELEMENT     // Can be instantiated from QML
    // ...
};

// Singleton type
class SingletonClass: public QObject {
    Q_OBJECT
    QML_ELEMENT
    QML_SINGLETON   // Only one instance, accessible globally
    // ...
};

// Uncreatable type (accessible but cannot be instantiated from QML)
class ReadOnlyData: public QObject {
    Q_OBJECT
    QML_ELEMENT
    QML_UNCREATABLE("ReadOnlyData cannot be created from QML")
    // ...
};
```

**Why**: Qt6 uses compile-time macro-based registration instead of runtime type registration. This eliminates runtime overhead, reduces binary size, and removes the need for plugin factory classes.

### Files Deleted

- `plugin.h` — No longer needed (replaced by `QML_ELEMENT` macro)
- `plugin.cpp` — No longer needed (replaced by `QML_ELEMENT` macro)
- `qmldir` — Auto-generated by `qt_add_qml_module()`
